### **READ (RETRIEVE)**

#### Get All Employees with Filters & Pagination
```javascript
const employees = await Employee.find(query)
    .sort({ [sortBy]: sortOrder })    // Sort by field (name, salary, date)
    .limit(limit)                      // Limit results per page
    .skip((page - 1) * limit);         // Skip for pagination
```
**MongoDB Operations Used:**
- `find()` - Returns array of matching documents
- `sort()` - Orders results (1 = ascending, -1 = descending)
- `limit()` - Limits number of results
- `skip()` - Skips N documents (used for pagination)

#### Get Single Employee by ID
```javascript
const employee = await Employee.findById(req.params.id);
```
**MongoDB Operation:** `db.employees.findOne({ _id: ObjectId("...") })`

#### Get Attendance Records for Employee
```javascript
const attendanceRecords = await Attendance.find({ employeeId });
```

#### Get Attendance with Date Range
```javascript
const attendance = await Attendance.find(query).sort({ date: -1 });
// Where query includes: { date: { $gte: startDate, $lte: endDate } }
```

#### Get Leave Requests with `.populate()` (JOIN)
```javascript
const leaves = await Leave.find(query)
    .populate("employeeId", "name email")  // Get employee details
    .populate("approvedBy", "name")        // Get approver details
    .sort({ createdAt: -1 });
```
**MongoDB Operation:** `$lookup` (similar to SQL JOIN)

## 4. QUERY OPERATORS

Your system uses these MongoDB query operators:

### **$regex - Text Search (Case Insensitive)**
```javascript
query.$or = [
    { name: { $regex: search, $options: "i" } },      // Search name
    { email: { $regex: search, $options: "i" } },     // Search email
    { phone: { $regex: search, $options: "i" } }      // Search phone
];
```
**Effect:** Finds documents where ANY field matches the search string

### **$gte / $lte - Range Queries**
```javascript
query.date = {
    $gte: new Date(startDate),   // Greater than or equal
    $lte: new Date(endDate)      // Less than or equal
};
```
**Effect:** Finds all records within a date range

## 5. AGGREGATION PIPELINE

### **Calculate Average Salary**
```javascript
const avgSalary = await Employee.aggregate([
    {
        $group: {
            _id: null,
            averageSalary: { $avg: "$salary" }
        }
    }
]);
```

**Breakdown:**
- `$group` - Groups documents
- `_id: null` - Groups ALL documents together
- `$avg: "$salary"` - Calculates average of salary field

### **Count by Department**
```javascript
const departments = await Employee.aggregate([
    {
        $group: {
            _id: "$department",
            count: { $sum: 1 }
        }
    },
    {
        $sort: { count: -1 }
    }
]);
```

**Breakdown:**
- `$group` by department name
- `$sum: 1` - Count documents in each group
- `$sort` - Orders by count descending

## 11. PAGINATION IMPLEMENTATION

```javascript
const page = parseInt(req.query.page) || 1;
const limit = parseInt(req.query.limit) || 10;

const total = await Employee.countDocuments(query);      // Total matching
const employees = await Employee.find(query)
    .skip((page - 1) * limit)      // Skip previous pages
    .limit(limit);                 // Get page size

// Response includes pagination info
pagination: {
    total,
    page,
    limit,
    pages: Math.ceil(total / limit)  // Total pages available
}
```

**Example:**
```
Page 1: skip 0, limit 10    (documents 1-10)
Page 2: skip 10, limit 10   (documents 11-20)
Page 3: skip 20, limit 10   (documents 21-30)
```

## 15. COMMON MONGODB COMMANDS USED

```javascript
// CREATE
db.collection.insertOne()      → new Model().save()
db.collection.insertMany()     → (multiple saves)

// READ
db.collection.find()           → Model.find()
db.collection.findOne()        → Model.findOne()
db.collection.findOneAndUpdate()→ Model.findByIdAndUpdate()

// UPDATE
db.collection.updateOne()      → Model.findByIdAndUpdate()
db.collection.updateMany()     → (bulk updates)

// DELETE
db.collection.deleteOne()      → Model.findByIdAndDelete()
db.collection.deleteMany()     → (bulk deletes)

// ANALYSIS
db.collection.countDocuments() → Model.countDocuments()
db.collection.distinct()       → Model.distinct()
db.collection.aggregate()      → Model.aggregate()

// SEARCH
db.collection.find({$regex})   → {$regex, $options: "i"}
db.collection.find({$gte})     → {$gte, $lte} range queries
```
